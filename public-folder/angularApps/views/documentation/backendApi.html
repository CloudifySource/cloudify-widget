To work with the backend API you will need our API Key.
You can see your API key once you log in to the widget.
Just go to Account section.

Once you have the API key, you need to add it to the Header request.

AccountUuid: api_key

GET: /backend/application/pool/status

returns the pool status.

Use the following CURL command to try it out now 
curl http://{{domain}}/backend/application/pool/status --header "AccountUuid: {{accountUuid}}"
{ "all":-1, "nonRemote":-1, "busyServers":-1, "nonBusyServers":-1, "minLimit":-1, "maxLimit":-1, "undergoingBootstrap":-1, "belowLimit":false }
Where -1 represents a number you will receive.
Most of the properties are deprecated by now, and soon will be ommitted from response,
The relevant fields are
  • nonBusyServers - represents the current pool size
  • undergoingBootstrap - represents the number of machines that will soon be available in the pool
  • busyServers - The servers currently used in the pool
